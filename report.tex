\begin{abstract}
This report details an experiment designed to intentionally overfit a Multi-Layer Perceptron (MLP) on the MNIST dataset and subsequently mitigate this overfitting using regularization techniques. The project is divided into two phases. Phase 1 demonstrates the phenomenon of overfitting using a high-capacity model without regularization. Phase 2 applies Data Augmentation and $L_{2}$ Regularization (Weight Decay) to improve the model's ability to generalize to unseen data.
\end{abstract}

\section{Introduction}
Overfitting is a fundamental challenge in machine learning where a model learns the training data too well, capturing noise and specific anomalies rather than underlying patterns. This leads to excellent performance on the training set but poor generalization to unseen validation or test data. This project explores overfitting empirically by constructing an overly complex neural network for the relatively simple MNIST digit classification task and then resolving the issue.

\section{Methodology}

\subsection{Dataset Preparation}
The project utilizes the MNIST dataset, which consists of $28 \times 28$ grayscale images of handwritten digits (0-9). The data is partitioned into:
\begin{itemize}
    \item \textbf{Training Set:} 50,000 images
    \item \textbf{Validation Set:} 10,000 images
    \item \textbf{Test Set:} 10,000 images
\end{itemize}

Two data loading strategies are employed:
\begin{enumerate}
    \item \textbf{Standard Loading:} Images are converted to PyTorch tensors and normalized to have a mean of 0.1307 and a standard deviation of 0.3081. This is used for all validation and test sets, and the training set in Phase 1.
    \item \textbf{Data Augmentation:} For Phase 2, the training set undergoes dynamic transformations during loading, including random rotations (up to 15 degrees) and random affine translations (up to 10\% horizontally and vertically). This artificially increases dataset diversity.
\end{enumerate}

\subsection{Model Architecture: OverfittingMLP}
To induce overfitting, a high-capacity Multi-Layer Perceptron (MLP) named \texttt{OverfittingMLP} is defined. The architecture is intentionally deep and extremely wide for this task:
\begin{itemize}
    \item \textbf{Input:} Flattened 784-dimensional vector ($28 \times 28$).
    \item \textbf{Hidden Layer 1:} Linear layer with 1024 neurons, followed by a ReLU activation.
    \item \textbf{Hidden Layer 2:} Linear layer with 1024 neurons, followed by a ReLU activation.
    \item \textbf{Hidden Layer 3:} Linear layer with 1024 neurons, followed by a ReLU activation.
    \item \textbf{Output Layer:} Linear layer with 10 neurons (logits for 10 classes).
\end{itemize}
This massive parameter count allows the network to easily memorize the 50,000 training examples.

\subsection{Training Procedure}
The model is trained using the Adam optimizer with a learning rate of 0.001 and Cross-Entropy Loss over 20 epochs. A batch size of 128 is used. The training loop calculates the loss and accuracy for both the training and validation sets at the end of each epoch. Furthermore, an early stopping mechanism is simulated by saving the model weights that achieve the lowest validation loss.

\section{Experiments and Results}

The execution is split into two distinct experimental phases.

\subsection{Phase 1: Overfitting the Model}
In the first phase, the model is trained with standard data loading (no augmentation) and no $L_{2}$ regularization (\texttt{weight\_decay=0.0}).

\paragraph{Expectations:} Given the model's large capacity, we expect the training accuracy to approach 100\% and the training loss to approach zero. However, the validation loss will likely plateau or even increase after a few epochs, indicating that the model is failing to generalize.

\paragraph{Results:} The outputs of this phase typically show a divergence between training and validation metrics. The script generates \texttt{phase\_1\_-\_overfitting.png}, visualizing this divergence.

\subsection{Phase 2: Resolving Overfitting}
In the second phase, a fresh instance of the exact same \texttt{OverfittingMLP} architecture is instantiated. This time, we apply two regularization techniques:
\begin{enumerate}
    \item \textbf{Data Augmentation:} The training data is subject to random rotations and shifts.
    \item \textbf{$L_{2}$ Regularization (Weight Decay):} A \texttt{weight\_decay} of $1 \times 10^{-4}$ is added to the Adam optimizer. This adds a penalty to the loss function proportional to the square of the magnitude of the model's weights, encouraging simpler models.
\end{enumerate}

\paragraph{Expectations:} The addition of noise to the input (augmentation) and constraints on the weights ($L_{2}$ penalty) will make it much harder for the model to memorize the training data. The training accuracy might be slightly lower than in Phase 1, but the validation and test accuracies should be significantly improved and more stable. The gap between training and validation curves should narrow.

\paragraph{Results:} The script evaluates this regularized model and outputs \texttt{phase\_2\_-\_regularization.png}, which visually demonstrates the effectiveness of these techniques in closing the generalization gap.

\section{Conclusion}
This project provides a practical demonstration of overfitting in deep learning. By deliberately constructing an overly complex model, we observe the failure of the model to generalize. Subsequently, by implementing standard regularization techniques—Data Augmentation and $L_{2}$ Regularization—we successfully mitigate the overfitting, resulting in a more robust model that performs better on unseen test data.
